# MSCS_634_Lab_3

## Lab Overview

This lab explores unsupervised machine learning through clustering analysis using the Wine Dataset from the Scikit-learn library. The primary objective is to implement and compare two popular clustering algorithms—K-Means and K-Medoids—to better understand how different clustering techniques group similar observations without using class labels during training.

The lab includes data preprocessing through feature standardization, implementation of both clustering algorithms, evaluation of clustering quality using the Silhouette Score and Adjusted Rand Index (ARI), and visualization of the resulting clusters using Principal Component Analysis (PCA). By comparing both numerical evaluation metrics and graphical representations, this lab demonstrates the strengths and limitations of each clustering approach.

---

## Dataset

The Wine Dataset is a well-known machine learning dataset provided by the Scikit-learn library. It contains **178 wine samples** described by **13 numerical chemical attributes**, with each sample belonging to one of **three different wine classes**.

Because the dataset contains multiple numerical features with different scales, z-score standardization was applied prior to clustering to ensure that all features contributed equally to the distance calculations used by both algorithms.

---

## Purpose of the Lab

The purpose of this lab was to:

- Load and explore the Wine Dataset.
- Standardize the dataset using z-score normalization.
- Implement the K-Means clustering algorithm with three clusters.
- Implement the K-Medoids clustering algorithm with three clusters.
- Evaluate clustering performance using the Silhouette Score and Adjusted Rand Index (ARI).
- Visualize and compare clustering results using Principal Component Analysis (PCA).
- Analyze the differences between K-Means and K-Medoids and determine when each algorithm is most appropriate.

---

## Clustering Results

The performance of both clustering algorithms was evaluated using two commonly used clustering metrics.

| Algorithm | Silhouette Score | Adjusted Rand Index (ARI) |
|-----------|-----------------:|--------------------------:|
| K-Means | **0.2849** | **0.8975** |
| K-Medoids | **0.2676** | **0.7411** |

The results indicate that **K-Means outperformed K-Medoids** on the Wine dataset. K-Means achieved both a higher Silhouette Score and a substantially higher Adjusted Rand Index, indicating that it generated more cohesive clusters while also more accurately recovering the true wine classes.

---

## Key Insights and Observations

Several important observations were made during this lab:

- Standardizing the dataset before clustering was essential because distance-based algorithms are sensitive to differences in feature scales.
- Both K-Means and K-Medoids successfully partitioned the Wine dataset into three clusters, reflecting the three underlying wine classes.
- K-Means produced more compact and well-separated clusters, as indicated by its higher Silhouette Score.
- The Adjusted Rand Index demonstrated that K-Means more closely matched the actual class labels than K-Medoids.
- PCA provided an effective two-dimensional visualization that made it easier to compare the clustering structures generated by both algorithms.
- Although K-Means performed better on this dataset, K-Medoids remains advantageous for datasets containing significant noise or outliers because it represents clusters using actual observations rather than calculated centroids.

## Challenges and Decisions

One challenge encountered during this lab involved implementing the K-Medoids algorithm in Google Colab. The originally intended `scikit-learn-extra` package was incompatible with the current Python and NumPy environment, resulting in import errors. To resolve this issue, an alternative K-Medoids implementation was used, allowing the clustering analysis to be completed successfully while maintaining the same evaluation methodology.

Another important decision was to use **Principal Component Analysis (PCA)** for visualization. Since the Wine dataset contains thirteen numerical features, reducing the data to two principal components allowed both clustering algorithms to be visualized clearly while preserving most of the dataset's variance.

Finally, both clustering algorithms were evaluated using identical preprocessing steps and the same performance metrics to ensure that the comparison between K-Means and K-Medoids was fair and consistent.
